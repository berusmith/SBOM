# SEC-002 — Dynamic + Isolated PoC Evidence

## Round 1 — via FastAPI endpoint (initial PoC)

```
[2] sending 499 bytes of billion-laughs payload (depth 4)
[3] response: HTTP 200 in 2.03s
```

**Initial interpretation (rev-1 — REJECTED)**:2-second response was attributed to entity expansion. **Round 2 isolated test below proves this was wrong** — actual XML parsing at depth 4 takes <1ms. The 2 seconds in the live-backend test was FastAPI request overhead (multipart parse + auth + admin lookup + etc.). XML expansion contributed ~1ms.

This file is now updated with Round 2 results. Including this honesty correction so audit trail shows where my initial interpretation was over-confident.

## Verdict (PARTIAL_DEFENSE_CONFIRMED)

**Existing defense layer**(Python 3.12 stdlib, NOT from this codebase):
- `expat 2.5+` includes an "amplification factor" check — total expansion / input size ratio.  Triggers at very high amplification.
- This blocks the **classic billion-laughs** depth-9 (10⁹) attack at parse time.

**Residual vulnerability**:
- depth 6 (1M expansion → 5 MB allocation per request) **passes through expat unmolested**.
- Single-worker uvicorn with 1 GB RAM ceiling → sustained barrage of depth-6 requests still produces DoS via memory pressure.
- Per-request CPU cost at depth 6 is 34 ms (parse only;FastAPI overhead adds ~2s).  At 30 RPS sustained you fill RAM faster than GC can free.
- Future Python version downgrades (running on 3.10 in CI / fallback hosts) would regress to no protection at all.

**Severity calibration result**(per user round-3 ask):
- **NOT bumped to High** — single-shot OOM not feasible on 3.12.
- **Stays at `severity_lan_only: Low / severity_if_public: Medium`**.
- Fix still required because:
  1. Depth 6 sustained DoS works (memory alloc + GC churn).
  2. expat protection is upstream;not under our control;version pinning is a maintenance smell.
  3. Pre-parse rejection of `DOCTYPE`/`ENTITY` is 2 lines, no perf cost on legitimate SBOMs.

## Affected endpoint clarification

- `POST /api/convert?target=...` — uses `xml.etree.ElementTree.fromstring` when input filename ends `.xml`. **AFFECTED**.
- `POST /api/releases/{id}/sbom` — `sbom_parser.py` is **JSON-only** (`json.loads`). **NOT an XML attack surface**.

`/api/convert` is router-level auth-required, so any viewer JWT is sufficient to reach the parser. No internet exposure today (LAN-only). Commercialisation exposes to all paying customers.
