## Severity model (per Q3 + Q5 user-provided rules)

Every finding in Phase 3/4 reports has **dual severity columns**:

```
| ID | severity_lan_only | severity_if_public | blocks_commercialization | compliance_impact |
```

Rules:
- **A**:if a finding violates SOC 2 Type II / ISO 27001 / GDPR DPIA / IEC 62443-4-1 (SDLC) when commercialised → bump `severity_if_public` one level + tag `blocks_commercialization: true`
- **B**:if finding only matters on public deployment (LAN-only neutralised) → `severity_lan_only` stays Low/Informational, `severity_if_public` per public threat model, still tag `blocks_commercialization: true` if applicable
- **C**:findings that hold even on LAN (e.g. plaintext secret in DB = defence-in-depth failure) → both columns get normal scoring; `blocks_commercialization` per control mapping
- `compliance_impact` column lists relevant controls (SOC 2 CC6.1, ISO 27001 A.8.24, GDPR Art. 32, IEC 62443-4-1 SI-1 etc.) so commercialisation gap analysis is one click away

**Pre-tagged severity examples (will appear in Phase 3 with full evidence):**
- X-Forwarded-For spoof on rate limit → `severity_lan_only: Low / severity_if_public: High` + `blocks_commercialization: true` (SOC 2 CC6.7)
- nginx security headers missing → `severity_lan_only: Informational / severity_if_public: Medium` + `blocks_commercialization: true`
- AlertConfig.webhook_url plaintext in DB → both cols get severity (defence-in-depth failure regardless of LAN)

## Note on the LAN-only assumption

LAN-only is **not zero-trust**. Same-subnet exposure to:
- IoT devices (smart appliances, cameras)
- Personal laptops / phones
- Anything on the home WiFi

Lateral-movement cost is low. Findings that "only matter on public deployment" still need eyes on if a threat model assumes a hostile device on the same subnet (CWE-940 — improper verification of source of communication).

## 2. 架構拓樸與信任邊界

```
                    Internet
                       │

        ┌────────┐     │
        │ Static │     │  /api/ → 反向代理
        │  /     │ ◀───┤
        │ /help/ │     │  / → React SPA(try_files fallback)
        └────────┘     │
                       ▼
              ┌─────────────────┐
              │ FastAPI uvicorn │  127.0.0.1:9100,**1 worker only**
              │  (launchd)      │
              └────┬────────────┘
                   │
    ┌──────────────┼──────────────┬──────────────────────┐
    ▼              ▼              ▼                      ▼
 SQLite/PG    OSV/NVD/EPSS    SMTP / Webhook        subprocess: Trivy / Syft / EMBA
 (本地檔案)   (HTTP outbound)  (SSRF 防護:私網黑名單)  (本地 CLI,執行用戶輸入路徑)
```

**信任邊界**(Phase 2 將為每個邊界產出 STRIDE):
1. **Internet ↔ nginx**:外部攻擊者
2. **nginx ↔ FastAPI**:已過 nginx,但 X-Forwarded-For 是反向代理唯一可信來源
3. **FastAPI ↔ DB**:本機 socket / 127.0.0.1:5432
4. **FastAPI ↔ External APIs**:OSV/NVD/etc.,Trust = "不會送入 attacker-controlled 資料"
5. **FastAPI ↔ Webhook outbound**:呼叫使用者填寫的 URL,有 SSRF 防護
6. **FastAPI ↔ subprocess**(Trivy/Syft/EMBA):工具讀使用者上傳的 zip / image ref
7. **Browser ↔ FastAPI**:JWT Bearer + API Token(`sbom_` 前綴)
8. **Multi-tenant**(Org A ↔ Org B):同一程序內,僅以 `org_scope` 邏輯隔離

## 5. 權限模型

### Authentication(三層,皆走 `core/deps.py:get_current_user`)
1. **JWT Bearer**(主)— HS256,8h TTL,jti claim 可被 RevokedToken 黑名單
2. **API Token**(`sbom_*` 前綴)— 雜湊存,scope read/write/admin
3. **OIDC SSO**(可選)— 走 IdP,callback 後同樣發 JWT,**透過 URL fragment 傳給前端**(防 Referer 落 log)

### Authorization
- **Role**:`admin` | `viewer`(`users.role`)
- **Scope**(API Token):`read`(只允許 GET)| `write`(GET/POST/PATCH/PUT)| `admin`(含 DELETE / 帳號 / Plan 操作)
- **Org scope**:`viewer` 只能看自己的 `org_id`;`admin` 看全部
- **Plan**:`starter` / `standard` / `professional`(`organization.plan`),透過 `core/plan.py:require_plan(feature)` dependency 守衛功能存取;不足回 **402**
- 守衛 helper:`require_admin`(`deps.py:59`)、`require_admin_scope`(`deps.py:69`)、`get_org_scope`(`deps.py:77`)
- Per-resource 守衛:`_assert_release_org`、`_assert_vuln_org`(在各 router 內,共 216 處 grep 點)— Phase 3 將窮舉是否每個非全域端點都有

### Session(JWT)生命週期
- 簽發:`create_access_token`(`security.py:56`)— `sub/role/exp/jti`,選填 `org_id` / `user_id`
- 解碼:`decode_token`(`security.py:71`)— 只接 HS256
- **登出**:寫 `RevokedToken(jti)` 黑名單(`auth.py:logout` 處理)
- **過期清除**:`main.py` 啟動時清除過期 RevokedToken(待 Phase 3 確認真的有跑)
- **無 refresh token**:8h TTL 後一律重登 — 流程簡單但有 UX trade-off

### Cookie / Session
- 平台**不用 cookie 存 token**(JWT 透過 `Authorization: Bearer` header)
- **OIDC state 用 cookie**(state cookie httpOnly + SameSite=Lax)— 待 Phase 3 確認設定
