### 🔬 Phase 2.4a: 韌體掃描基礎
**改動範圍**: 3 個後端端點 + 1 個前端頁面 (大)  
**工作時長**: 5-7 小時  
**前置條件**: 需要 EMBA 框架（Linux 環境）

**具體項目**:

   - `POST /api/firmware/upload` — 接收 .bin/.img/.zip

   - `GET /api/firmware/scans/{id}` — 檢查進度 + 結果

2. 前端:
   - FirmwareUpload 頁面 (拖放上傳)
   - 掃描進度條 (WebSocket 即時更新)
   - 結果預覽 + 匯入為 Release 功能

3. 後端服務:
   - `firmware_service.py` — 呼叫 EMBA CLI
   - 解析 EMBA JSON 輸出 → 轉為 SBOM 元件

**預期效果**: 可上傳韌體 → 自動生成 SBOM + 掃描漏洞
